How this works

    All the necessary data is gathered via webscraping, but since i'm not up to pay for a website hosting, i'm using github pages which is free, although it comes with some limitations such as being a static page. A static page is quite a big problem to keep this calendar up to date, so my work around was to output my webscraping to a json, which is put into this page's javascript. This way, all i have to do is to run my python webscraper every a certain amount of time and commit the changed json to this repository, which is a task that i can put my computer to automatically execute once it is turned on. I chose to update every month so that the repository don't get polluted by automatic commits.

    The whole visual part of this page was done entirely with raw javascript and css.
    The webscraper is also in this page's repository if you want to check it out.
